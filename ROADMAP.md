# Roadmap

## Before public Recipe submission

- [x] Capture and review a real 800×480 TRMNL render.
- [x] Receive or document Jazz Lineup's response to the limited, attributed reuse request. (Reuse blessed by the maintainer for community-Recipe submission, with visible attribution to jazzlineup.com and the understanding that the feed is not a formal API.)
- [ ] Tag the first stable Recipe release after the above checks.

## Afterwards

- [ ] Add a separate small-display Recipe, rather than shrinking the complete board.
- [ ] Add a documented visual-regression workflow once a stable renderer/fixture is available.
- [ ] Consider additional display preferences only when the source data supports them reliably.
- [ ] Explore a city-configurable v2. The maintainer notes the same feed pattern exists for other cities — LA (`events-la.json`), Chicago, SF, Paris, and London, with more coming — so a single configurable Recipe could serve any of them. (Confirm each city's exact feed URL before relying on it.)

This is deliberately not a plan to mirror or archive Jazz Lineup's calendar.
